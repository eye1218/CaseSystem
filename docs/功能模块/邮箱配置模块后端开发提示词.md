# 邮箱发送者配置模块后端开发提示词

## 模块目标
- 提供管理员可维护的邮件发送者配置（Mail Sender Config）。
- 支持新增、编辑、启停、测试发送、最近测试结果记录。
- 供任务模块引用，任务执行时校验发送者有效性。

## 数据模型建议
- `MailSenderConfig`
  - `id`
  - `sender_name`
  - `sender_email`
  - `auth_account`
  - `auth_password`（敏感字段，仅存储，不在响应中返回）
  - `smtp_host`
  - `smtp_port`
  - `security_type`：`SSL | TLS | STARTTLS`
  - `status`：`ENABLED | DISABLED`
  - `latest_test_status`：`SUCCESS | FAILED | null`
  - `latest_test_at`
  - `latest_test_error_summary`
  - 审计字段（created_by / updated_by / timestamps）
- `MailSenderAuditLog`（最小审计落库）
  - actor、action、sender_id、summary、created_at

## API 边界
- `GET /api/v1/mail-senders`
- `POST /api/v1/mail-senders`
- `GET /api/v1/mail-senders/{id}`
- `PATCH /api/v1/mail-senders/{id}`
- `POST /api/v1/mail-senders/{id}/status`
- `POST /api/v1/mail-senders/{id}/test`

## 规则约束
- 仅 `ADMIN` 可访问和操作。
- 本期不提供删除接口；发送者只能停用。
- 编辑时 `auth_password` 可不传，不传则保留原密码。
- `sender_email` 与 `auth_account` 允许不一致。
- 测试发送必须使用固定模板，不允许自定义测试正文。
- 测试失败需分类并写入摘要（认证失败/连接失败/发送失败）。
- 错误信息、日志、响应中不得明文回显 `auth_password`。

## 与任务模块的集成要求
- Email 任务模板可绑定 `sender_config_id`。
- 任务执行前需校验：
  - 发送者存在
  - 发送者状态为 `ENABLED`
- 不满足则任务实例进入 `FAILED`，并记录失败原因。

## 本期不实现
- 发件人删除能力
- SMTP 健康巡检与自动告警
- 多发送者负载均衡或自动切换
- 邮件发送历史中心页面
