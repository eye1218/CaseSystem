# 预览部署

当前项目的预览环境部署方式已经固定为：

- 本地通过 `rsync` 同步代码
- 远端通过项目内 `.venv` 安装依赖
- 使用 `systemd + uvicorn` 常驻运行
- 如需处理延迟 / 定时 / 超时 Event，可额外拉起 `celery worker + celery beat`
- 预览端口默认使用 `8010`
- 标准入口固定为 `./scripts/deploy_preview.sh`

当前已验证目标机：

- 主机：`192.168.20.142`
- 用户：`root`
- 目录：`/root/workspace/CaseSystem`
- 服务名：`casesystem-preview`
- 健康检查地址：[http://192.168.20.142:8010/healthz](http://192.168.20.142:8010/healthz)

## 一键部署

项目根目录执行：

```bash
./scripts/deploy_preview.sh
```

脚本会执行以下步骤：

1. 运行 `backend/tests`
2. 自动解析本地 Python 环境，优先使用当前工作区 `.venv`，在 `git worktree` 场景下自动回退到主工作区共享 `.venv`
3. 在本地测试前执行 `pip install -e '.[dev]'`，把共享 `.venv` 同步到当前 `pyproject.toml`
4. 使用 `rsync` 同步仓库到远端
5. 在远端创建或复用 `.venv`
6. 执行 `.venv/bin/python -m pip install -e '.[dev]'`
7. 写入 `/etc/systemd/system/casesystem-preview.service`
8. 执行 `systemctl daemon-reload`
9. 重启并拉起 `casesystem-preview`
10. 调用远端 `http://127.0.0.1:8010/healthz` 做带重试的健康检查

部署完成后默认输出预览地址。

## 可覆盖参数

脚本支持通过环境变量覆盖默认值：

```bash
REMOTE_HOST=192.168.20.142 \
REMOTE_USER=root \
REMOTE_DIR=/root/workspace/CaseSystem \
SERVICE_NAME=casesystem-preview \
APP_PORT=8010 \
./scripts/deploy_preview.sh
```

如果本地虚拟环境不在默认位置，可以显式指定：

```bash
LOCAL_PYTHON=/abs/path/to/.venv/bin/python ./scripts/deploy_preview.sh
```

如果只想跳过“本地依赖同步”这一步：

```bash
SYNC_LOCAL_ENV=0 ./scripts/deploy_preview.sh
```

如果只想跳过本地测试：

```bash
RUN_TESTS=0 ./scripts/deploy_preview.sh
```

如果需要在预览环境中启用 Celery Worker / Beat 处理延迟、定时和超时 Event，可以显式传入：

```bash
CELERY_ENABLE_SERVICES=1 \
CELERY_BROKER_URL=redis://127.0.0.1:6379/0 \
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/1 \
CELERY_EVENT_SWEEP_INTERVAL_SECONDS=30 \
./scripts/deploy_preview.sh
```

说明：

- 不传 `CELERY_ENABLE_SERVICES=1` 时，脚本只部署 API 服务，适合本地即时 Event 验证。
- 开启 Celery 服务后，脚本会额外部署：
  - `${SERVICE_NAME}-celery-worker`
  - `${SERVICE_NAME}-celery-beat`
- 延迟、定时、超时 Event 依赖 Beat 定时扫描 `events.pending` 队列，未开启时不会自动派发。

## 首次部署前提

远端需要满足：

- 可以通过 SSH 连接
- 具备 `python3`
- 具备 `systemd`
- 当前用户有权限写入 `/etc/systemd/system`

本地需要满足：

- 已创建当前工作区 `.venv`，或主工作区共享 `.venv`
- 已安装开发依赖
- 可使用 `rsync`

## 常用排查

查看服务状态：

```bash
ssh root@192.168.20.142 'systemctl status casesystem-preview --no-pager'
```

查看服务日志：

```bash
ssh root@192.168.20.142 'journalctl -u casesystem-preview -n 200 --no-pager'
```

确认远端服务文件：

```bash
ssh root@192.168.20.142 'systemctl cat casesystem-preview'
```

手动重启：

```bash
ssh root@192.168.20.142 'systemctl restart casesystem-preview'
```

## 当前约束

- 目标机的 `80/443` 已被其他服务占用，所以预览暂时固定走 `8010`
- 当前预览服务未接入反向代理，适合内部验收，不适合作为正式生产入口
- 标准部署流程就是执行 `./scripts/deploy_preview.sh`；除非明确说明，否则后续部署默认都按这个脚本执行
