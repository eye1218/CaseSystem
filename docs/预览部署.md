# 预览部署

当前项目的预览环境部署方式已经固定为：

- 本地通过 `rsync` 同步代码
- 远端通过项目内 `.venv` 安装依赖
- 使用 `systemd + uvicorn` 常驻运行
- 预览端口默认使用 `8010`

当前已验证目标机：

- 主机：`192.168.2.170`
- 用户：`root`
- 目录：`/root/workspace/CaseSystem`
- 服务名：`casesystem-preview`
- 健康检查地址：[http://192.168.2.170:8010/healthz](http://192.168.2.170:8010/healthz)

## 一键部署

项目根目录执行：

```bash
./scripts/deploy_preview.sh
```

脚本会执行以下步骤：

1. 运行 `backend/tests`
2. 使用 `rsync` 同步仓库到远端
3. 在远端创建或复用 `.venv`
4. 执行 `.venv/bin/python -m pip install -e '.[dev]'`
5. 写入 `/etc/systemd/system/casesystem-preview.service`
6. 执行 `systemctl daemon-reload`
7. 重启并拉起 `casesystem-preview`
8. 调用远端 `http://127.0.0.1:8010/healthz` 做带重试的健康检查

部署完成后默认输出预览地址。

## 可覆盖参数

脚本支持通过环境变量覆盖默认值：

```bash
REMOTE_HOST=192.168.2.170 \
REMOTE_USER=root \
REMOTE_DIR=/root/workspace/CaseSystem \
SERVICE_NAME=casesystem-preview \
APP_PORT=8010 \
./scripts/deploy_preview.sh
```

如果只想跳过本地测试：

```bash
RUN_TESTS=0 ./scripts/deploy_preview.sh
```

## 首次部署前提

远端需要满足：

- 可以通过 SSH 连接
- 具备 `python3`
- 具备 `systemd`
- 当前用户有权限写入 `/etc/systemd/system`

本地需要满足：

- 已创建项目内 `.venv`
- 已安装开发依赖
- 可使用 `rsync`

仓库中保留了当前预览服务模板：

## 常用排查

查看服务状态：

```bash
ssh root@192.168.2.170 'systemctl status casesystem-preview --no-pager'
```

查看服务日志：

```bash
ssh root@192.168.2.170 'journalctl -u casesystem-preview -n 200 --no-pager'
```

确认远端服务文件：

```bash
ssh root@192.168.2.170 'systemctl cat casesystem-preview'
```

手动重启：

```bash
ssh root@192.168.2.170 'systemctl restart casesystem-preview'
```
