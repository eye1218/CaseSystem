# Figma 回收站提示词

本文档整理 SOC 7x24 工单系统中“回收站页”的 Figma 高保真设计提示词，可直接提供给 Figma AI 或其他界面生成工具使用。

## 提示词

```text
请为一个 “SOC 7x24 工单系统” 设计企业级 Web 后台中的“回收站页 / Recycle Bin”，面向安全运营中心（SOC）场景。该页面用于展示被软删除的工单，强调可追溯、可恢复、权限严格、操作谨慎。页面风格必须专业、冷静、可信、克制，偏系统治理后台，不要做成普通文件回收站或轻量化列表。

一、页面定位

这是管理员专用页面，用于管理被软删除的工单。

页面目标：
- 清晰查看哪些工单已被软删除
- 快速判断删除时间、删除人、删除前关键信息
- 支持筛选、检索和审计追溯
- 支持恢复工单
- 强调谨慎操作和留痕

二、权限要求

页面必须明确体现：
- 仅管理员可见
- 非管理员不可见该菜单
- 无权限访问时展示明确的权限受限状态
- 回收站中的数据仍属于受保护业务对象，受对象级访问控制约束

三、布局结构

请采用企业后台高密度列表页布局：
- 顶部页面标题区
- 顶部说明卡片或提示栏
- 左侧高级筛选区或可折叠筛选抽屉
- 中央高密度表格列表区
- 右侧详情侧栏或抽屉预览区

四、页面内容

1. 顶部说明区
展示：
- 页面标题：回收站 / Recycle Bin
- 简短说明：此处展示已软删除工单，可用于审计追溯与恢复
- 管理风险提示：恢复会将工单返回正常业务流程，需保留恢复日志

2. 统计概览卡片
展示：
- 当前回收站工单数量
- 今日新增删除数量
- 最近 7 天删除趋势
- 待恢复重点工单数量

3. 筛选与检索
必须支持：
- 关键词搜索（工单 ID、标题）
- 按删除时间范围筛选
- 按删除人筛选
- 按分类筛选
- 按删除前主状态筛选
- 按优先级筛选
- 按风险分区间筛选

4. 回收站表格
表格字段建议包含：
- 工单 ID
- 标题
- 删除前主状态
- 删除前子状态
- 分类
- 来源
- 优先级
- 风险分
- 原处理人
- 原池子
- 删除时间
- 删除人
- 删除原因摘要（如有）
- 操作

要求：
- 高密度但可读
- 支持固定表头
- 支持固定首列
- 支持排序
- 支持列管理
- 支持保存视图
- 状态标签、优先级、风险分组件需与主系统保持一致

五、操作设计

回收站页的核心操作为：
- 查看删除前摘要信息
- 恢复工单

操作要求：
- “恢复工单”应为主操作之一，但必须有明确确认步骤
- 恢复动作需强调会恢复到正常业务流程
- 恢复后需要有系统留痕提示
- 如果项目尚未定义永久删除规则，不要把“彻底删除”设计成默认操作
- 如果展示“彻底删除”占位，也必须为高风险次级操作，并带明显风险警示

六、详情侧栏 / 抽屉

点击表格行或“查看详情”后，可在右侧侧栏或抽屉中展示删除前摘要信息：
- 工单标题
- 描述摘要
- 删除前主状态 / 子状态
- 删除前责任人 / 池子
- 分类
- 来源
- 优先级
- 风险分
- 创建时间
- 最后更新时间
- 删除时间
- 删除人
- 删除原因摘要
- 最近一次关键操作日志摘要

要求：
- 详情侧栏用于快速判断，不需要替代完整工单详情页
- 如需查看更完整历史，可提供“查看原工单历史快照”的次级入口
- 恢复按钮可放在侧栏头部，但要有确认流程

七、视觉与交互要求

页面风格应偏系统治理后台：
- 冷静
- 严谨
- 有审计感
- 强调受控操作

状态视觉建议：
- 已软删除使用低饱和风险色或中性警示色
- 恢复操作使用明确但克制的强调色
- 高风险操作避免过度显眼但要足够警示

八、主题与国际化

页面必须支持：
- 中文 / English 双语切换
- Light / Dark Theme 双主题

要求：
- 表格、状态标签、筛选器、抽屉在亮暗主题下都具备良好可读性
- 英文文案长度通常更长，请预留列宽和按钮宽度空间

九、典型状态

请在设计稿中体现以下状态：
- 正常列表状态
- 空状态（暂无已删除工单）
- 加载中
- 错误状态
- 无权限状态
- 恢复确认状态
- 恢复成功反馈状态

十、输出要求

请输出高保真回收站页设计，并包含：
- 页面主布局
- 筛选区
- 高密度表格
- 右侧详情侧栏 / 抽屉
- 恢复确认交互
- 空状态 / 无权限状态 / 恢复成功状态
- 中英文双语表现
- Light / Dark Theme 双主题表现

请优先输出可落地前端实现的企业级后台页面，不要为了装饰性牺牲信息密度与审计可读性。
```
