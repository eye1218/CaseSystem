# Figma 任务模块提示词

本文档整理 SOC 7x24 工单系统中“任务模块”的 Figma 高保真设计提示词，可直接提供给 Figma AI 或其他界面生成工具使用。

## 提示词

```text
请为一个 “SOC 7x24 工单系统” 设计企业级 Web 后台中的“任务模块 / Tasks Module”，面向安全运营中心（SOC）场景。该模块由两个页面组成：

1. 任务工作台 / Tasks
2. 配置中心下的任务模板管理 / Configuration > Task Templates

整体风格必须延续企业级安全运营后台的视觉语言：
- 专业
- 冷静
- 可信
- 克制
- 高信息密度
- 偏系统治理后台，不要做成营销型 SaaS 或轻量化看板

请不要把任务模块设计成“流程编排平台”或“自动化机器人中心”。本期任务模块只处理 Event 触发后的任务实例查看与任务模板维护。

一、整体范围

请输出两张核心页面的高保真设计，并保证属于同一套设计系统：

1. 任务工作台 / Tasks
- 主导航一级菜单
- 只对内部角色可见
- 用于查看任务实例、筛选、打开详情抽屉、对失败任务执行手动重试

2. 任务模板管理 / Configuration > Task Templates
- 配置中心二级页面
- 仅管理员可见
- 用于维护 Email / Webhook 两类任务模板
- 采用列表 + 右侧侧边编辑面板

二、权限与可见性

页面设计必须明确体现：
- 内部角色可见 Tasks 页面
- 客户角色不显示 Tasks 菜单
- Task Templates 页面仅管理员可见
- 无权限访问时应有明确受限态

补充规则：
- 客户不能查看任务执行日志
- 客户不能执行失败重试
- 模板维护不向非管理员开放

三、页面一：任务工作台 / Tasks

1. 页面定位

该页面是任务实例运行态工作台，不是模板页，也不是流程编排页。

页面目标：
- 快速查看 Event 触发后产生的任务实例
- 用高密度表格承载大量任务
- 通过筛选快速定位失败、运行中、指定模板或指定工单关联的任务
- 通过右侧详情抽屉查看单个任务的完整执行上下文
- 对失败任务执行人工重试

2. 布局结构

请采用企业后台高密度工作台布局：
- 顶部标题区
- 概览统计区
- 筛选器区
- 中央任务表格
- 右侧非模态详情抽屉

要求：
- 列表固定采用表格视图，不设计卡片流
- 详情抽屉打开后，表格区域仍应保留可见宽度
- 右侧抽屉为非模态，不要让整个页面失去上下文

3. 顶部信息区

展示：
- 页面标题：任务工作台 / Task Execution Workspace
- 简要说明：集中查看 Event 触发的任务实例与执行结果
- 刷新按钮
- 中英文切换入口
- Light / Dark Theme 切换入口

4. 统计概览

建议展示：
- 任务总数
- 运行中
- 成功
- 失败

风格要求：
- 信息密度高，但视觉秩序清晰
- 不要夸张大卡片
- 强调“可读性”和“状态分层”

5. 筛选区

必须支持以下筛选：
- 关键词搜索：任务 ID / 任务名称 / 工单 ID
- 任务类型：Email / Webhook
- 任务状态：Pending / Running / Success / Failed / Cancelled
- 来源 Event
- 任务模板
- 工单 ID
- 开始时间范围
- 失败快捷筛选：Failed only

要求：
- 筛选器适合高频使用
- 英文文案变长后不应挤压失真
- 日期范围需清晰可读

6. 表格设计

任务列表必须使用高密度表格，列固定为：
- 任务 ID
- 任务名称
- 类型
- 来源 Event
- 关联工单
- 目标摘要
- 状态
- 开始 / 结束时间
- 最近结果
- 操作

表格要求：
- 支持固定表头
- 高密度但不可拥挤到无法扫描
- 状态标签要与系统整体视觉一致
- Email / Webhook 类型要有明确区分
- 最近结果适合展示简短摘要，如 Accepted、Template render failed、smtp down
- 操作列至少包含 Detail；失败任务额外包含 Retry

7. 详情抽屉

点击表格行或 Detail 后，在右侧非模态抽屉中展示：
- 来源 Event
- 任务模板快照
- 渲染结果摘要
- 第三方响应摘要
- 错误信息
- 执行日志时间线
- Retry 操作入口

要求：
- 不设计独立详情页
- 抽屉头部要保留任务名称、状态、任务 ID
- 日志时间线强调审计感与执行链路
- 若任务失败，错误信息区域需要视觉聚焦但不过度惊悚

8. 关键状态

请体现以下典型状态：
- 正常列表状态
- Failed only 筛选状态
- 打开详情抽屉状态
- 失败任务重试确认状态
- 空状态
- 加载中
- 错误状态
- 无权限状态

四、页面二：任务模板管理 / Task Templates

1. 页面定位

该页面位于配置中心中，是管理员维护任务模板的专用页面。

本期仅支持两类任务模板：
- Email
- Webhook

请不要扩展：
- 自动重试
- 多步骤编排
- 取消任务
- 第三种任务类型

2. 布局结构

请采用“左侧表格清单 + 右侧侧边编辑面板”的布局：
- 页面标题区
- 顶部说明
- 搜索与筛选
- 模板列表表格
- 右侧侧边编辑面板

要求：
- 列表与编辑要同屏协作
- 右侧编辑面板既可承载创建态，也可承载编辑态
- 不跳转独立详情页

3. 列表筛选

支持：
- 搜索：模板名称 / 模板 ID
- 任务类型筛选
- 状态筛选：Active / Inactive

4. 模板表格

建议字段：
- 模板
- 类型
- 引用模板
- 目标 / 收件摘要
- 状态
- 更新时间
- 操作

展示规则：
- Email 模板展示 To / Cc / Bcc 摘要
- Webhook 模板展示固定目标说明：当前 Event 工单上下文
- 状态支持 Enable / Disable

5. 右侧编辑面板

公共字段固定为：
- 名称
- 任务类型
- 引用模板
- 启停状态
- 描述

Email 模板额外字段：
- To 收件规则
- Cc 收件规则
- Bcc 收件规则

收件规则要求：
- 多行规则编辑
- 每行仅支持三种来源：Custom Email / Current Handler / Role Members
- 如果是 Role Members，需要选择角色
- 如果是 Custom Email，需要输入邮箱地址

Webhook 模板额外要求：
- 明确展示“目标对象 = 来源 Event 的当前工单上下文”
- 不要提供 URL / Method / Headers / Body 覆盖输入
- 要让用户一眼看出：请求内容完全由引用模板决定

6. 页面说明信息

请在设计中明确提示：
- Email 任务模板负责收件规则编排
- Webhook 任务模板不允许覆盖请求内容
- 保存成功后会生成真实 task_template_id，可被 Event 规则绑定

7. 关键状态

请体现：
- 创建新模板状态
- 编辑 Email 模板状态
- 编辑 Webhook 模板状态
- Active / Inactive 切换确认状态
- 表单校验错误状态
- 空列表状态
- 无权限状态

五、语言、主题与视觉要求

页面必须支持：
- 中文 / English 双语
- Light / Dark Theme 双主题

要求：
- 表格、筛选器、状态标签、抽屉、侧边面板在亮暗主题下均具备高可读性
- 英文文本通常更长，请预留布局空间
- 暗色主题偏夜间值守场景，避免纯黑
- 亮色主题偏白天办公与配置管理场景

六、输出要求

请输出高保真的任务模块设计稿，并至少包含：
- 任务工作台主页面
- 任务工作台失败任务详情抽屉
- 失败重试确认状态
- 任务模板管理页面
- Email 模板编辑态
- Webhook 模板编辑态
- 无权限状态
- 空状态
- 中英文双语表现
- Light / Dark Theme 双主题表现

请优先输出可落地前端实现的企业级后台页面，不要为了装饰感牺牲信息密度、表格可读性和审计清晰度。
```
