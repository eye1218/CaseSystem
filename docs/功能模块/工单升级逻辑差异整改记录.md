# 工单升级逻辑差异整改记录

本文件用于记录《[工单升级逻辑](./工单升级逻辑.md)》与当前系统实现之间发现的不一致项，以及本次整改的处理结果。

## 2026-03-15

### 已整改差异 1：建单接口仍残留“直接归属个人”旧模式

- 对应文档：
  - BF-01
  - BF-02
  - 3.3 归属池子
  - 3.4 归属个人
- 差异描述：
  - 文档要求工单创建后必须进入池子，未指定时默认进入 `T1_POOL`
  - 当前接口契约仍接受 `assignment_mode=self`，与文档定义不一致
- 整改结果：
  - 后端建单请求 schema 已收口为仅支持 `unassigned | pool`
  - 后端服务层已对不支持的 `assignment_mode` 返回 `422`
  - 前端类型与相关测试已同步更新

### 已整改差异 2：创建页对“领取后池子是否保留”的说明错误

- 对应文档：
  - 3.5 领取
  - 3.6 分配
  - 6. AC-06 / AC-07 / AC-08 / AC-11
- 差异描述：
  - 文档明确要求领取或分配后，`current_pool_code` 仍保留原值
  - 创建页说明文案此前写成“被领取后当前池子清空”
- 整改结果：
  - 创建页说明文案已改为“填写当前处理人，但保留原池子”

### 已整改差异 3：`ticket.source` 的对外语义不符合文档

- 对应文档：
  - 2.1 第 2 条
  - BF-01 / BF-02
- 差异描述：
  - 文档将工单来源定义为 `API / INTERNAL / CUSTOMER`
  - 当前系统对外会直接暴露 `SIEM / EDR` 等上游检测器值
- 整改结果：
  - 工单接口对外返回的 `source` 已统一收口到 `API / INTERNAL / CUSTOMER`
  - 任务模板上下文里的 `ticket.source` 也已同步收口
  - 上游检测器来源继续保留在原始告警与上下文信息中，不再混入工单来源字段
