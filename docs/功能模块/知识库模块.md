# 知识库模块开发需求文档

## 1. 模块概述

知识库模块用于沉淀、维护和复用与工单处理相关的标准化知识内容，作为系统内部的 Markdown 知识资产管理入口存在。

本模块在系统中的定位是：

1. 为内部人员提供统一的知识库查看、创建、编辑、删除与点赞入口；
2. 通过“关联工单类别”的方式，与工单详情页形成关联展示；
3. 为工单处理过程中的经验复用提供结构化支撑。

本模块与以下内容直接相关：

* 工单模块：知识库通过“关联工单类别”与工单产生关联；
* 权限模块：内部人员、作者本人、管理员具备不同操作权限；
* 内容展示模块：知识库正文统一使用 Markdown 格式存储与展示。

---

## 2. 建设目标

本期建设目标如下：

1. 提供知识库列表页，以卡片形式展示全部知识库；
2. 支持按工单类别筛选知识库，筛选为单选；
3. 支持按统一规则排序知识库：先置顶、再按点赞数倒序、再按更新时间倒序；
4. 支持知识库详情预览；
5. 支持从知识库模块新建知识库；
6. 支持从工单详情页跳转新建知识库，并自动带入工单类别；
7. 支持知识库编辑；
8. 支持知识库删除；
9. 支持真实点赞与取消点赞能力；
10. 支持管理员置顶与取消置顶；
11. 在工单详情页展示“相关知识库”标识和关联知识库；
12. 全部知识库正文统一采用 Markdown。

## 4. 角色与权限

### 4.1 普通内部人员

#### 可查看

1. 可查看全部知识库；
2. 可查看知识库列表页与详情页；
3. 可查看知识库的工单类别名称标签；
4. 可在工单详情页查看相关知识库。

#### 可执行

1. 可新建知识库；
2. 可编辑自己创建的知识库；
3. 可删除自己创建的知识库；
4. 可在知识库详情页对任意可见知识库点赞或取消点赞。

#### 不可执行

1. 不可编辑他人创建的知识库；
2. 不可删除他人创建的知识库；
3. 不可置顶或取消置顶；
4. 不可在列表页直接点赞或取消点赞。

#### 页面表现要求

1. 查看他人创建的知识库时，编辑按钮置灰；
2. 查看他人创建的知识库时，删除按钮置灰。

### 4.2 作者本人

作者本人属于普通内部人员的一个特殊场景，额外具备以下权限：

1. 可编辑自己创建的知识库；
2. 可删除自己创建的知识库。

### 4.3 管理员

#### 可查看

1. 可查看全部知识库；
2. 可查看全部相关知识库展示区域。

#### 可执行

1. 可新建任意知识库；
2. 可编辑任意知识库；
3. 可删除任意知识库；
4. 可置顶任意知识库；
5. 可取消置顶任意知识库；
6. 可点赞或取消点赞任意可见知识库。

#### 不可执行

1. 无本模块内的内容权限限制，但仍受系统统一身份认证与访问控制约束。

### 4.4 外部客户 / 非内部用户

1. 不可见知识库模块；
2. 不可访问知识库列表、详情、新建、编辑、删除、点赞等任何能力；
3. 不可查看工单详情页中的相关知识库内容。

## 5. 核心概念说明

### 5.1 知识库

知识库是本模块的核心对象，表示一篇可被内部人员查看和复用的 Markdown 文档。

### 5.2 工单类别关联

知识库不直接与具体工单做主关联，而是通过“关联工单类别”与工单建立关系。

规则为：

* 一篇知识库只能关联一个工单类别；
* 当工单当前类别与知识库关联的工单类别一致时，视为该知识库与该工单相关；
* 工单详情页基于该规则展示相关知识库。

### 5.3 工单类别名称

知识库页面中展示的类别名称，不是知识库内可手工编辑的自由文本，而是来自其所关联的工单类别主数据。

因此：

* 不允许直接编辑“类别名称文本”；
* 允许在有权限时修改“关联工单类别”；
* 若工单类别主数据名称发生变更，知识库展示的类别名称同步变化。

### 5.4 置顶

置顶是管理员可操作的排序优先标记，不是独立状态流转。置顶后的知识库在排序时优先于非置顶知识库。

### 5.5 点赞关系

点赞关系是用户与知识库之间的单条行为关系。规则为：

* 同一用户对同一知识库最多只能存在 1 条有效点赞关系；
* 支持取消点赞；
* 点赞数为该知识库的有效点赞关系总数。

### 5.6 相关知识库

相关知识库是指在工单详情页中，按“工单当前类别 = 知识库关联工单类别”规则匹配出来的知识库集合。

### 5.7 相关知识库标识

“相关知识库”标识只在工单详情页的相关知识库区域展示，用于提示当前展示内容为与该工单类别相关的知识内容。知识库自身页面不展示该标识，只展示工单类别名称标签。

## 6. 业务规则

### 6.1 关联规则

1. 一篇知识库只能关联一个工单类别；
2. 知识库与工单的关联依据为：工单当前类别与知识库关联工单类别一致；
3. 知识库详情页和列表页展示工单类别名称标签；
4. 工单详情页展示“相关知识库”标识；
5. 从工单详情页新建知识库时，应自动带入当前工单类别。

### 6.2 类别规则

1. 工单类别使用系统统一枚举；
2. 知识库关联的类别必须来自该枚举；
3. 不允许直接编辑知识库上的类别名称文本；
4. 允许通过修改“关联工单类别”改变知识库的类别归属；
5. 工单类别主数据名称变更后，所有关联该类别的知识库展示名称同步更新。

### 6.3 权限规则

1. 普通内部人员可查看全部知识库；
2. 普通内部人员仅可编辑和删除自己创建的知识库；
3. 管理员可编辑、删除任意知识库；
4. 管理员可置顶和取消置顶任意知识库；
5. 所有内部人员都可对任意可见知识库点赞或取消点赞；
6. 非内部用户不可访问本模块。

### 6.4 点赞规则

1. 点赞是实际功能，不是静态展示字段；
2. 点赞与取消点赞仅能在知识库详情页执行；
3. 列表页仅展示点赞数，不提供点赞操作入口；
4. 详情页需展示“我已点赞”状态；
5. 同一用户对同一知识库最多只能存在 1 条点赞关系；
6. 点赞数实时更新；
7. 列表页和详情页的点赞数展示需保持一致；
8. 删除知识库时，其关联点赞关系一并物理删除。

### 6.5 排序规则

统一排序优先级如下：

1. 先按是否置顶排序；
2. 再按点赞数倒序排序；
3. 再按更新时间倒序排序。

该排序规则至少适用于：

* 知识库列表页；
* 工单详情页中的相关知识库展示。

### 6.6 删除规则

1. 本期删除为物理删除；
2. 删除后知识库本体与关联点赞关系一并删除；
3. 删除后前台全部不可见；
4. 删除后旧链接访问返回“知识库不存在或已删除”；
5. 删除后工单详情页中的相关知识库同步移除；
6. 删除不需要二次确认；
7. 本期不提供恢复能力。

### 6.7 页面行为规则

1. 非作者普通内部人员查看他人知识库时，编辑按钮置灰；
2. 非作者普通内部人员查看他人知识库时，删除按钮置灰；
3. 作者本人查看自己知识库时，编辑、删除按钮可用；
4. 管理员查看任意知识库时，编辑、删除、置顶相关操作可用；
5. 新建成功和编辑成功后，进入知识库详情预览页。

### 6.8 已解决的潜在歧义

针对“类别名称不能编辑”与“类别可变更”的表述，本文按以下方式统一解释：

* 禁止手工修改类别名称文本；
* 允许修改知识库关联的工单类别；
* 知识库页面上展示的类别名称，来自当前关联的工单类别主数据。

## 7. 业务流程

### 7.1 正常流程：浏览知识库

1. 内部人员进入知识库模块；
2. 系统校验其内部访问权限；
3. 返回知识库列表；
4. 前端按卡片形式展示知识库；
5. 用户可按工单类别单选筛选；
6. 系统按统一排序规则返回结果；
7. 用户点击卡片进入知识库详情页。

### 7.2 正常流程：从知识库模块新建知识库

1. 内部人员进入知识库模块；
2. 点击“新建知识库”；
3. 打开新建页面；
4. 用户填写标题、关联工单类别、Markdown 正文等内容；
5. 提交保存；
6. 系统校验必填项与权限；
7. 保存成功后进入知识库详情预览页。

### 7.3 正常流程：从工单详情页新建知识库

1. 内部人员在工单详情页点击“新建知识库”；
2. 系统跳转至知识库新建页；
3. 系统自动带入当前工单类别；
4. 用户填写其余内容并提交；
5. 系统保存成功后进入知识库详情预览页；
6. 新建完成后，该知识库按其关联类别参与相关知识库匹配展示。

### 7.4 正常流程：编辑知识库

1. 用户进入知识库详情页；
2. 若用户具备编辑权限，则编辑按钮可用；
3. 用户进入编辑页；
4. 修改允许编辑的内容；
5. 提交保存；
6. 系统校验权限与字段合法性；
7. 保存成功后进入详情预览页。

### 7.5 正常流程：删除知识库

1. 用户进入知识库详情页；
2. 若用户具备删除权限，则删除按钮可用；
3. 用户执行删除；
4. 系统物理删除知识库；
5. 系统物理删除其关联点赞关系；
6. 删除后该知识库从所有前台展示中消失；
7. 旧链接访问时返回“知识库不存在或已删除”。

### 7.6 正常流程：点赞 / 取消点赞

1. 用户进入知识库详情页；
2. 查看当前点赞状态；
3. 未点赞时可执行点赞；
4. 已点赞时可执行取消点赞；
5. 系统按“同一用户同一知识库最多一条点赞关系”规则处理；
6. 点赞数实时更新；
7. 列表页与详情页展示保持一致。

### 7.7 正常流程：管理员置顶 / 取消置顶

1. 管理员进入知识库详情页或管理入口；
2. 对目标知识库执行置顶或取消置顶；
3. 系统更新其置顶状态；
4. 后续列表与相关知识库区域按统一排序规则重新展示。

### 7.8 异常流程：无权限编辑或删除

1. 普通内部人员访问他人创建的知识库详情页；
2. 系统根据当前用户与创建人判断无编辑/删除权限；
3. 编辑按钮置灰；
4. 删除按钮置灰；
5. 若前端被绕过直接发起请求，后端仍应拒绝操作。

### 7.9 异常流程：知识库不存在或已删除

1. 用户访问知识库详情链接；
2. 系统查询不到对应知识库；
3. 返回“知识库不存在或已删除”；
4. 不返回知识库实际内容。

### 7.10 边界情况：工单类别名称变更

1. 系统中的工单类别主数据名称变更；
2. 所有关联该工单类别的知识库，在展示时统一显示新类别名称；
3. 不保留旧名称展示。

### 7.11 边界情况：工单类别变更

1. 某工单当前类别发生变化；
2. 工单详情页的相关知识库应按新类别重新匹配；
3. 原类别下的相关知识库不再展示。

### 7.12 状态变化流程

本期不设计草稿、发布、归档等内容状态流转。知识库仅存在以下业务标记变化：

1. 置顶 / 非置顶；
2. 已点赞 / 未点赞；
3. 存在 / 已物理删除。

## 8. 功能详细说明

### 8.1 知识库列表页

#### 功能目的

提供全部知识库的统一浏览入口。

#### 触发方式 / 入口

系统菜单中的知识库模块入口。

#### 输入信息

1. 当前用户身份；
2. 类别筛选条件；
3. 排序条件采用系统内置统一规则。

#### 系统处理逻辑

1. 校验用户是否具备内部访问权限；
2. 查询全部可见知识库；
3. 应用类别筛选；
4. 按统一排序规则排序；
5. 以卡片形式返回结果。

#### 输出结果

知识库卡片列表。

#### 限制条件

1. 列表页不提供点赞操作；
2. 列表页仅支持类别单选筛选；
3. 列表页不实现全文搜索。

#### 异常处理要求

1. 无权限时拒绝访问；
2. 无数据时展示空状态；
3. 查询异常时返回统一错误提示。

---

### 8.2 知识库卡片展示

#### 功能目的

用卡片形式展示知识库的核心信息，便于浏览和进入详情。

#### 触发方式 / 入口

知识库列表加载后自动展示。

#### 输入信息

单篇知识库的基础数据。

#### 系统处理逻辑

至少展示以下信息：

1. 标题；
2. 工单类别名称标签；
3. 点赞数；
4. 更新时间。

#### 输出结果

可点击进入详情的知识库卡片。

#### 限制条件

1. 卡片不提供点赞按钮；
2. 卡片不展示“相关知识库”标识。

#### 异常处理要求

字段缺失时不得导致页面结构损坏，应有容错显示。

---

### 8.3 类别筛选

#### 功能目的

按工单类别缩小知识库范围。

#### 触发方式 / 入口

知识库列表页筛选区。

#### 输入信息

一个工单类别值。

#### 系统处理逻辑

1. 接收单个类别筛选值；
2. 过滤关联该工单类别的知识库；
3. 返回筛选结果。

#### 输出结果

筛选后的知识库卡片列表。

#### 限制条件

1. 仅支持单选；
2. 不支持多选；
3. 不支持用户自定义类别。

#### 异常处理要求

非法类别值应被拒绝，不得导致异常返回。

---

### 8.4 统一排序

#### 功能目的

保证知识库在不同展示区域排序一致。

#### 触发方式 / 入口

知识库列表页、工单详情页相关知识库区域。

#### 输入信息

无用户自定义排序输入，使用系统内置规则。

#### 系统处理逻辑

按以下顺序排序：

1. 是否置顶；
2. 点赞数倒序；
3. 更新时间倒序。

#### 输出结果

已排序的知识库结果集。

#### 限制条件

本期不提供用户自定义排序方式切换。

#### 异常处理要求

排序字段异常时应进行系统容错，避免列表不可用。

---

### 8.5 知识库详情预览页

#### 功能目的

展示知识库完整内容，并承载点赞、编辑、删除等操作入口。

#### 触发方式 / 入口

点击知识库卡片进入。

#### 输入信息

知识库标识、当前用户身份。

#### 系统处理逻辑

1. 查询知识库详情；
2. 返回 Markdown 正文并渲染；
3. 展示工单类别名称标签；
4. 展示点赞数及“我已点赞”状态；
5. 根据权限控制编辑、删除、置顶相关按钮状态。

#### 输出结果

知识库详情页。

#### 限制条件

1. 详情页是唯一的点赞与取消点赞入口；
2. “相关知识库”标识不在知识库自身页面展示。

#### 异常处理要求

知识库不存在或已删除时，返回明确提示。

---

### 8.6 新建知识库

#### 功能目的

新增一篇知识库。

#### 触发方式 / 入口

1. 知识库模块中的新建入口；
2. 工单详情页中的新建入口。

#### 输入信息

1. 标题；
2. 关联工单类别；
3. Markdown 正文。

#### 系统处理逻辑

1. 校验当前用户是否具备创建权限；
2. 校验必填字段；
3. 保存知识库；
4. 保存成功后进入详情预览页。

#### 输出结果

新建成功的知识库详情页。

#### 限制条件

1. 一篇知识库只能关联一个工单类别；
2. 正文必须为 Markdown；
3. 从工单详情页进入时需自动带入工单类别。

#### 异常处理要求

1. 必填项缺失时禁止保存；
2. 非法工单类别值禁止保存；
3. 无权限时拒绝创建。

---

### 8.7 编辑知识库

#### 功能目的

修改已有知识库内容或其关联工单类别。

#### 触发方式 / 入口

知识库详情页编辑按钮。

#### 输入信息

1. 知识库标识；
2. 可编辑字段的新值。

#### 系统处理逻辑

1. 校验当前用户是否具备编辑权限；
2. 加载知识库原始内容；
3. 允许修改标题、正文、关联工单类别等可编辑字段；
4. 不允许手工编辑类别名称文本；
5. 保存成功后进入详情预览页。

#### 输出结果

更新后的知识库详情页。

#### 限制条件

1. 普通内部人员仅可编辑自己创建的知识库；
2. 管理员可编辑任意知识库。

#### 异常处理要求

1. 越权编辑必须被拒绝；
2. 非法工单类别值必须被拒绝；
3. 已删除知识库不得编辑。

---

### 8.8 删除知识库

#### 功能目的

移除一篇知识库及其关联点赞数据。

#### 触发方式 / 入口

知识库详情页删除按钮。

#### 输入信息

知识库标识。

#### 系统处理逻辑

1. 校验当前用户是否具备删除权限；
2. 对知识库执行物理删除；
3. 同步物理删除关联点赞关系；
4. 删除后从所有展示区域移除。

#### 输出结果

删除成功结果。

#### 限制条件

1. 不进行二次确认；
2. 不提供恢复能力。

#### 异常处理要求

1. 越权删除必须被拒绝；
2. 已删除对象重复删除应返回明确失败结果；
3. 删除后旧链接访问应提示“知识库不存在或已删除”。

---

### 8.9 点赞 / 取消点赞

#### 功能目的

提供真实的内容认可行为。

#### 触发方式 / 入口

知识库详情页点赞按钮。

#### 输入信息

当前用户、知识库标识。

#### 系统处理逻辑

1. 查询当前用户是否已点赞；
2. 未点赞时创建点赞关系；
3. 已点赞时删除点赞关系；
4. 实时刷新点赞数和“我已点赞”状态。

#### 输出结果

最新点赞状态与点赞数。

#### 限制条件

1. 仅详情页可操作；
2. 同一用户同一知识库最多一个点赞关系。

#### 异常处理要求

1. 已删除知识库不可点赞；
2. 越权访问不可点赞；
3. 并发情况下不得出现重复点赞关系。

---

### 8.10 管理员置顶 / 取消置顶

#### 功能目的

通过管理员干预改变知识库排序优先级。

#### 触发方式 / 入口

管理员可见的操作入口。

#### 输入信息

知识库标识、目标置顶状态。

#### 系统处理逻辑

1. 校验管理员身份；
2. 更新目标知识库置顶状态；
3. 后续展示统一按内置排序规则返回。

#### 输出结果

置顶状态更新成功。

#### 限制条件

1. 仅管理员可操作；
2. 允许多个知识库同时置顶；
3. 支持取消置顶。

#### 异常处理要求

非管理员操作必须被拒绝。

---

### 8.11 工单详情页相关知识库展示

#### 功能目的

在工单处理场景中展示与当前工单类别相关的知识库。

#### 触发方式 / 入口

工单详情页加载时。

#### 输入信息

当前工单的类别。

#### 系统处理逻辑

1. 获取工单当前类别；
2. 匹配所有关联该类别的知识库；
3. 按统一排序规则排序；
4. 展示“相关知识库”标识；
5. 展示匹配出的知识库列表。

#### 输出结果

工单详情页中的相关知识库区域。

#### 限制条件

1. 仅依据工单当前类别匹配；
2. 不按工单 ID 做主关联；
3. 知识库自身页面不展示“相关知识库”标识。

#### 异常处理要求

无匹配结果时应展示空状态而非异常。

## 9. 接口 / 数据交互要求

### 9.1 核心输入信息

本模块至少需要以下核心输入信息：

1. 当前用户身份信息；
2. 知识库标识；
3. 标题；
4. 关联工单类别；
5. Markdown 正文；
6. 置顶状态变更请求；
7. 点赞 / 取消点赞操作请求；
8. 筛选条件中的工单类别值。

### 9.2 核心输出信息

系统至少需要对前端返回以下内容：

1. 知识库标识；
2. 标题；
3. Markdown 正文或渲染所需原文；
4. 关联工单类别标识；
5. 工单类别名称；
6. 点赞数；
7. 当前用户是否已点赞；
8. 是否置顶；
9. 创建人；
10. 更新时间；
11. 当前用户是否具备编辑权限；
12. 当前用户是否具备删除权限；
13. 当前用户是否具备置顶操作权限。

### 9.3 必填字段

新建或编辑时，至少以下字段应为必填：

1. 标题；
2. 关联工单类别；
3. Markdown 正文。

### 9.4 系统生成字段

以下字段应由系统生成或维护，不允许由前端自由指定：

1. 知识库标识；
2. 创建人；
3. 创建时间；
4. 更新时间；
5. 点赞数；
6. 当前用户点赞状态；
7. 置顶状态；

### 9.5 字段格式与取值限制

1. 关联工单类别必须来自系统工单类别枚举；
2. 一篇知识库只能关联一个工单类别；

### 9.6 需要预留扩展的字段能力

本期不实现，但建议设计时保留扩展空间：

1. 内容摘要；
2. 审计相关字段；
3. 版本相关字段；
4. 更多展示排序维度；
5. 统计类字段。

## 10. 验收标准

### 10.1 列表与展示

1. 内部人员可进入知识库列表页；
2. 列表页以卡片形式展示知识库；
3. 卡片至少能正确展示标题、工单类别名称标签、点赞数、更新时间；
4. 列表页不显示点赞操作按钮；
5. 非内部用户不可访问列表页。

### 10.2 类别筛选

1. 用户可按工单类别进行单选筛选；
2. 筛选结果仅包含关联该类别的知识库；
3. 非法类别值不会导致系统报错或越权返回数据。

### 10.3 统一排序

1. 置顶知识库始终排在非置顶知识库之前；
2. 同为置顶或同为非置顶时，按点赞数倒序排序；
3. 点赞数相同的情况下，按更新时间倒序排序；
4. 工单详情页相关知识库区域使用相同排序规则。

### 10.4 新建知识库

1. 内部人员可从知识库模块创建知识库；
2. 内部人员可从工单详情页跳转创建知识库；
3. 从工单详情页创建时，系统能自动带入当前工单类别；
4. 标题、关联工单类别、Markdown 正文缺失时不得保存成功；
5. 新建成功后进入知识库详情页。

### 10.5 编辑知识库

1. 作者本人可编辑自己创建的知识库；
2. 管理员可编辑任意知识库；
3. 非作者普通内部人员查看他人知识库时，编辑按钮置灰；
4. 前端绕过后发起越权编辑请求，后端必须拒绝；
5. 编辑成功后进入知识库详情页；
6. 编辑时不允许手工编辑类别名称文本；
7. 编辑时允许变更关联工单类别。

### 10.6 删除知识库

1. 作者本人可删除自己创建的知识库；
2. 管理员可删除任意知识库；
3. 非作者普通内部人员查看他人知识库时，删除按钮置灰；
4. 前端绕过后发起越权删除请求，后端必须拒绝；
5. 删除为物理删除；
6. 删除后关联点赞关系一并删除；
7. 删除后知识库不再出现在列表页、详情页、工单详情页相关知识库区域；
8. 删除后旧链接访问返回“知识库不存在或已删除”；
9. 删除操作不要求二次确认。

### 10.7 点赞 / 取消点赞

1. 详情页可对知识库执行点赞；
2. 详情页可对已点赞知识库执行取消点赞；
3. 同一用户对同一知识库重复点赞不会产生多条有效点赞关系；
4. 点赞数实时更新；
5. “我已点赞”状态正确展示；
6. 列表页与详情页点赞数保持一致；
7. 已删除知识库不可点赞。

### 10.8 置顶 / 取消置顶

1. 管理员可置顶任意知识库；
2. 管理员可取消置顶任意知识库；
3. 允许多个知识库同时置顶；
4. 非管理员无法进行置顶操作；
5. 置顶状态变化后，排序结果应立即体现。

### 10.9 工单详情页相关知识库

1. 工单详情页展示“相关知识库”标识；
2. 仅展示关联工单类别与当前工单类别一致的知识库；
3. 工单当前类别变化后，相关知识库展示结果同步变化；
4. 无匹配结果时展示空状态；
5. 知识库自身页面不展示“相关知识库”标识。

### 10.10 工单类别名称变化

1. 工单类别名称变更后，所有关联该类别的知识库展示名称同步变化；
2. 不保留旧类别名称展示。

## 11. 待确认问题

以下问题仍会影响具体页面或交互落地，需进一步确认：

1. 从工单详情页跳转到知识库新建页后，自动带入的工单类别是否允许用户在新建页改为其他类别；
2. 管理员置顶后的知识库，是否需要在列表页和详情页显示明显的“已置顶”视觉标识；
3. 删除成功后，前端页面的默认跳转目标是返回知识库列表页，还是返回上一页；
4. 工单详情页中的相关知识库区域，是展示全部匹配结果，还是仅展示前 N 条并支持查看更多。

## 12. 给研发的实现提醒

1. 不要把“类别名称”实现成知识库上的自由文本字段；当前正确口径是“知识库关联工单类别，类别名称来自工单类别主数据”；
2. 不要把知识库与具体工单做成主关联关系；本期主关联逻辑是按工单类别匹配；
3. 不要在列表页实现点赞交互；点赞和取消点赞只能在详情页执行；
4. 不要把删除实现成逻辑删除；本期明确要求物理删除，并同步删除点赞关系；
5. 不要默认增加删除二次确认；该规则已明确为不需要；
6. 前端按钮置灰不是最终权限控制，后端必
7. 须做严格鉴权；
8. 编辑与删除权限必须基于“是否作者本人 / 是否管理员”判断，不能仅依赖前端控制；
9. 列表页与工单详情页的排序规则必须统一，避免用户看到不同顺序；
10. 工单类别名称变更后，知识库展示名称必须同步，不应复制一份静态名称造成数据漂移；
11. 当前仅支持一篇知识库关联一个工单类别，研发不要自行扩展为多选关联；
12. 置顶不是独立生命周期状态，不要额外引入复杂状态机；
13. 已删除知识库的旧链接返回语义必须明确，统一为“知识库不存在或已删除”，不要返回模糊错误。
