# 工单 Event → 邮件模板 → 邮件发送 → 任务状态 全链路测试方案

---

## 1. 文档目的

本文档用于验证以下完整链路是否能够稳定、正确地跑通：

**工单业务动作触发 Event → Event 命中任务模板 → 任务实例生成 → 邮件模板渲染 → 邮件发送 → 任务状态流转与记录**

本方案重点覆盖：

1. 工单触发 Event
2. Event 的即时触发、提前触发、延后触发
3. 响应超时类 Event
4. 邮件模板渲染
5. 单收件人、多收件人、角色人员作为收件人
6. 邮件发送成功 / 失败
7. 任务状态流转
8. 失败后的可定位性与可恢复性

## 3. 模糊概念补充说明

---

### 3.1 全链路通过的判定标准

**收到邮件，不等于全链路通过。**

一个测试场景只有在以下条件同时满足时，才可判定为通过：

1. 工单动作真实发生
2. 对应 Event 被正确触发
3. 正确生成任务实例
4. 邮件模板正确渲染
5. 邮件正确发送到目标收件人
6. 任务状态正确流转
7. 日志与结果可追踪

---

### 3.2 提前触发 / 延后触发 的含义

提前触发、延后触发是指：

基于某个定义好的业务基准时刻，Event 并不是在基准时刻本身触发，而是在其之前或之后触发。

示例：

- 基准时刻：某个定时业务时刻 / SLA 截止时刻 / 计划触发时刻
- 提前触发：在该基准时刻到来前 `${EVENT_ADVANCE_OFFSET}` 触发
- 延后触发：在该基准时刻到来后 `${EVENT_DELAY_OFFSET}` 触发

本测试方案不假设技术实现方式，只验证最终行为是否正确：

1. 是否在正确时间点触发
2. 是否仅触发一次
3. 是否进入后续任务与邮件链路

---

### 3.3 响应超时 Event 的含义

响应超时 Event 指：

工单在规定的响应 SLA 时间内，未进入“已响应”或系统定义的响应达成状态，触发超时类 Event。

本方案测试重点不是 SLA 计算算法本身，而是：

1. 响应超时是否能正确触发 Event
2. Event 触发后是否能进入邮件任务链路
3. 任务状态是否正确记录

---

### 3.4 多收件人的含义

多收件人指单次邮件任务中存在多个目标邮箱。常见来源包括：

1. 手工配置多个固定邮箱
2. 当前处理人 + 固定邮箱
3. 某角色下全部人员邮箱

本方案验证重点：

1. 所有目标收件人是否被正确解析
2. 邮件是否实际发送给全部目标收件人
3. 其中一个收件人配置异常时，系统如何表现

---

### 3.5 角色人员作为收件人的含义

角色人员作为收件人，指邮件接收对象不是写死的邮箱，而是系统中某个角色下的全部有效成员。

本方案验证重点：

1. 角色成员是否被正确展开为邮箱列表
2. 无邮箱成员是否如何处理
3. 角色下多人时是否全部收到
4. 角色无人时任务是否失败或拒绝执行

---

### 3.6 任务状态流转的判定规则

任务状态至少要符合以下规则：

1. 新生成任务实例时，初始状态应为 `PENDING`
2. 开始执行时应进入 `RUNNING`
3. 成功完成时应进入 `SUCCESS`
4. 模板渲染失败、收件人解析失败、SMTP 发送失败时应进入 `FAILED`
5. 不允许原失败实例直接改成 `SUCCESS`
6. 重试应生成新的任务实例，不覆盖原失败实例

## 8. 统一核验点

---

### 8.1 工单层核验

1. 工单动作是否真实发生
2. 工单状态是否符合预期
3. 工单操作日志是否存在

---

### 8.2 Event 层核验

1. 是否命中正确的 Event
2. Event 是否在正确时刻触发
3. Event 是否仅触发一次
4. Event 是否绑定正确任务模板

---

### 8.3 任务实例层核验

1. 是否生成任务实例
2. 是否生成了正确数量的任务实例
3. 状态是否按 `PENDING → RUNNING → SUCCESS / FAILED` 流转
4. 是否记录失败原因
5. 重试是否生成新实例

---

### 8.4 模板层核验

1. 是否命中正确模板
2. `subject` 是否渲染正确
3. `body` 是否渲染正确
4. 模板变量是否来自正确业务上下文
5. 渲染失败时是否返回明确信息

---

### 8.5 收件人层核验

1. 收件人是否解析正确
2. 多收件人是否全部展开
3. 角色收件人是否按角色成员展开
4. 无邮箱、非法邮箱、空收件人是否按规则处理

---

### 8.6 SMTP / 邮件发送层核验

1. SMTP 是否收到发送请求
2. 邮件是否真正送达目标邮箱或 SMTP 捕获器
3. 主题、正文、收件人是否正确
4. 发送失败时是否可区分认证失败、连接失败、地址失败等问题

---

## 9. 执行顺序建议

1. 先验证 SMTP 基础连通
2. 再验证邮件模板渲染成功
3. 再执行 TC-001 / TC-002 / TC-003 主链路冒烟
4. 再执行 TC-004 响应超时
5. 再执行 TC-005 / TC-006 提前 / 延后
6. 再执行 TC-007 / TC-008 多收件人 / 角色收件人
7. 再执行 TC-009 / TC-010 异常链路
8. 最后执行 TC-011 / TC-012 / TC-013 / TC-014 恢复与边界场景
