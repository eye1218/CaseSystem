# Figma 提示词：配置中心 - 邮箱发送者配置模块

## 目标
- 基于现有 CaseSystem 管理后台视觉风格，输出邮箱发送者配置页面。
- 页面路径：`/configuration/mail-senders`，仅管理员访问。
- 核心布局：左侧高密度表格 + 右侧非模态编辑抽屉。

## 页面结构
- 顶部：面包屑说明、主标题、说明文案、`刷新` 与 `新增发送者` 按钮。
- 筛选栏（卡片）：关键词搜索、状态筛选（全部/启用/停用）。
- 列表表格（固定列）：
  - 发送者（名称 + ID）
  - 邮箱（sender_email + auth_account）
  - 状态（Badge）
  - 最近测试结果（成功/失败/未测试）
  - 最近测试时间
  - 更新时间
  - 操作（编辑、启用/停用）
- 右侧抽屉：
  - 模式一：创建发送者
  - 模式二：编辑发送者
  - 表单字段：sender_name、sender_email、auth_account、auth_password、smtp_host、smtp_port、security_type、status
  - 编辑态密码输入为空时表示不修改
  - 测试发送区（仅编辑态可见）：输入测试邮箱、执行测试、显示最近一次测试结果摘要

## 视觉与风格约束
- 沿用项目现有 Tailwind 样式语义：
  - 卡片 `rounded-2xl border border-slate-200 bg-white dark:*`
  - 表格头部使用 `text-[11px] uppercase tracking-*`
  - 状态与结果使用圆角 badge + 淡色背景
- 保持高密度表格，不引入卡片瀑布流布局。
- 右侧抽屉保持固定宽度，不使用全屏弹窗。

## 关键交互状态
- 默认列表（有数据）
- 列表空状态（筛选后无数据）
- 创建抽屉（未保存，不允许测试）
- 编辑抽屉（可测试发送）
- 测试成功状态
- 测试失败状态（展示错误摘要）

## 双语与主题
- 中英文都要可读，避免文案溢出。
- light / dark 两套主题均需截图。

## 捕获建议（Figma MCP）
- 至少捕获 4 个状态用于评审：
  - 列表默认状态（light）
  - 编辑抽屉 + 测试成功（light）
  - 编辑抽屉 + 测试失败（dark）
  - 创建抽屉（中文）
